# Data Mastery Challenge — Project Plan

This document outlines the working plan for the Data Mastery Challenge: a 180-day ultralearning experiment focused on financial analytics, predictive modeling, and applied data science.

The plan is intentionally structured, but not rigid. The goal is steady, visible progress through deliberate practice, project-based learning, and public documentation.

I chose financial analytics because it often includes elements of data science, statistics, and machine learning, which I am very interested in. I also chose it because in this current economy, it seems more important than ever to have a firm grasp on finances, and the broader picture of how the financial industry functions.

## Challenge Dates

**Start:** June 15, 2026  
**Initial End Date:** December 12, 2026  
**Duration:** 180 days

## Core Focus

This challenge focuses on applied data work in a financial context.

The main areas of study and practice are:

- SQL for analytical querying
- Python for data cleaning, analysis, and modeling
- Statistics and probability
- Predictive modeling
- Forecasting
- Credit risk analysis
- Fraud and anomaly detection
- Customer/member analytics
- Financial dashboards and reporting
- Data quality and validation
- Model evaluation and explainability
- AI-assisted analytical workflows
- Technical writing and documentation

## Guiding Rule

Every major topic should produce something visible.

That means each concept should eventually become one or more of the following:

- SQL query
- Python notebook
- Model
- Dashboard
- Written explanation
- Case study
- Technical note
- Project README
- Blog post
- Retrospective

The intention of this project is to learn through building, avoiding the "tutorial hell" trap many tech people fall into.

## Project Roadmap

The challenge is organized around six monthly projects. Each project is designed to stand on its own while also contributing to a broader financial analytics portfolio.

### Month 1 — Credit Risk Profile & Loan Portfolio Explorer

**Goal:** Build a financial analytics project that explores a loan portfolio and identifies borrower, loan, and portfolio-level risk patterns.

**Focus Areas:**

- Credit risk vocabulary
- Loan portfolio analysis
- SQL analysis
- Python exploratory data analysis
- Risk segmentation
- Data visualization
- Business-facing documentation

**Possible Outputs:**

- Loan portfolio analysis notebook
- SQL queries for risk segmentation
- Risk profile report
- Data dictionary
- Portfolio visualizations
- Project README

**Key Questions:**

- Which borrower segments appear riskiest?
- What variables are associated with delinquency or default?
- How is risk distributed across the portfolio?
- What data quality issues affect the analysis?
- What would a risk or lending team want to monitor?

- What limitations would matter in a real financial setting?

## Weekly Structure

Each week follows a flexible rhythm.

### Monday — Concept & Planning

- Study the main concept for the week
- Write brief notes
- Define the week’s deliverable
- Update the project README or weekly log

### Tuesday — Data Work

- Acquire, clean, or structure data
- Write SQL queries
- Build data dictionaries
- Profile data quality issues

### Wednesday — Modeling / Analysis

- Build or improve the analysis
- Create features
- Train or evaluate models
- Generate charts or metrics

### Thursday — Communication

- Turn analysis into explanation
- Build visuals or dashboard components
- Write findings
- Draft business recommendations

### Friday — Polish & Portfolio

- Clean code
- Improve documentation
- Update GitHub
- Draft a blog post or project note
- Record lessons learned

### Weekend — Deep Work Block

- Finish larger project components
- Resolve blockers
- Publish weekly log
- Prepare the next week’s focus

## Weekly Metrics

Each week, I will track:

- Focused study/build hours
- SQL/DataLemur/StrataScratch problems solved
- Python/modeling reps completed
- GitHub commits
- Project milestones completed
- Notes written
- Documentation updates
- Blockers encountered
- Lessons learned
- Next week’s focus

- 5 focused study/build hours
- 5 SQL or data problems

- 1 weekly log

### Target Weekly Targets

- 8–10 focused study/build hours
- 10 SQL or data problems

- 1 visible project milestone
- 1 documentation update

### Stretch Weekly Targets

- 12+ focused study/build hours
- 15+ SQL or data problems
- 1 major feature or analysis milestone
- 1 blog post, project note, or walkthrough
- 1 polished visual, model, or report component

## Monthly Metrics

Each month, the goal is to complete:

- 1 project
- 1 polished project README
- 1 business-facing summary or memo
- 1 analysis/modeling notebook
- 1 visual report, dashboard, or screenshot set
- 1 monthly retrospective
- 30–50 SQL/data practice problems
- 1 longer public reflection, if possible

## Project Definition of Done

A monthly project is considered complete when it has:

- A clear business problem
- A documented dataset
- Clean analysis or modeling workflow
- Reproducible code or notebooks
- Key findings
- Business recommendations
- Limitations
- Next steps
- A polished README
- At least one visual output
