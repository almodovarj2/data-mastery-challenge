# The Data Mastery Challenge

### What is it?

The Data Mastery Challenge is a 180-day ultralearning experiment. The goal is to become a more complete data professional through deliberate practice, project-based learning, and publicly documenting everything I learn and build along the way.

### Why?

In 2026, the bar for data professionals has risen dramatically. The skills that made data professionals valuable just a few years ago have become commoditized. To stand out now, you need to offer something **more**. Being good, or even great, is no longer enough. You have to be *exceptional*.

I've been a long-time fan of Scott H. Young’s *Ultralearning* and his MIT Challenge, as well as Cal Newport’s work on deep work, craft, and focused intensity. For that reason, I've decided to create my own ultralearning/deep work challenge to see what kind of progress I can make in half a year.

### The Goal

This is my public commitment to push the limits of what one person can achieve with deliberate, intense practice. I plan to document the journey rigorously on GitHub and my blog, sharing code, projects, reflections, struggles, and wins.

While rigid structure is not the point, the areas I plan to cover over the next 6 months are:

- Advanced SQL
- Python for data analysis
- Power BI / business intelligence
- Data visualization
- Statistics
- Regression analysis
- Machine learning fundamentals
- Data modeling
- Data quality
- Analytics engineering concepts
- AI-assisted data workflows
- Technical writing and documentation
- End-to-end data projects

My goal is not to collect certs, or passively consume information. The goal is to build skill through repetition and actually *creating* things with the skills I learn. Learn, then build. Learn, then build.

### What I’m Building Toward

By the end of the first 180 days, I want to have:

- Solved a large volume of SQL and data interview-style problems
- Built multiple Power BI dashboards
- Completed several Python data analysis projects
- Built practical statistics and regression projects
- Completed at least a few machine learning projects
- Created end-to-end data projects from raw data to final insight
- Written clear documentation for every major project
- Published regular progress updates
- Built a public body of work that shows real improvement

### Tracking

This repository will serve as the central hub for the challenge.

It may include:

- Weekly logs
- Monthly reviews
- Study notes
- SQL practice
- Python notebooks
- Dashboard screenshots
- Project documentation
- Reflections
- Resource notes
- Final retrospectives

My blog will be used for longer reflections, essays, and updates.

### Challenge Dates

**Start:** June 15, 2026  
**End Date:** December 12, 2026  
**Duration:** 180 days

If the first 180 days go well, I may extend the challenge into a second phase focused on deeper specialization and more advanced projects.

### Final Thoughts

This challenge is not about proving I can learn everything in six months.

It is about raising my standard.

It is about becoming harder to ignore.

It is about approaching data work as a craftman would. As something to study, practice, build, refine, and strive toward mastery.
